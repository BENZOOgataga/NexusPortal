# CONTEXT.md

## Purpose

Nexus Portal is an internal command interface for a Star Citizen organization.

Its purpose is to provide a structured, secure, and lore-consistent operational environment where members interact through hierarchy, authority, and controlled information flow.

This is not a community website.
This is not a marketing page.
This is not a social platform.

It is an internal system designed to reflect how a serious UEE-era organization would operate.

## What we are building

We are building a diegetic command terminal.

The platform must feel like an official RSI or UEE operational tool from the 2950+ era.  
Every design and engineering decision must reinforce this perception.

The system should:
- Reflect military or corporate discipline  
- Emphasize hierarchy and chain of command  
- Treat information as controlled and permission-bound  
- Prioritize clarity, structure, and traceability  

## Design identity

The interface must feel like:
- A classified internal terminal  
- A strategic operations console  
- A secure organizational backbone  

It must not feel like:
- A generic SaaS dashboard  
- A modern startup admin panel  
- A gaming guild website  

## Structural philosophy

The architecture must support:

- Strict authority modeling  
- Role-based access control as a foundational principle  
- Auditability for sensitive actions  
- Long-term modular expansion  

Permissions are not an add-on.
They are the backbone of the system.

## Tone and narrative consistency

The tone must remain:

- Formal  
- Corporate or military  
- In-universe  
- Technically precise  

No playful language.
No modern internet tone.
No casual UX patterns.

## Long-term direction

Nexus Portal should evolve into a unified internal command system.

It must remain coherent, modular, and aligned with its original vision:
A secure, authoritative, in-universe operational control platform for a Star Citizen organization.
